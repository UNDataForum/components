[`ProfilePreview`](/components/profile-preview)s in a grid.

## System props

`Badges` supports [space props](https://styled-system.com/api#space) and
[color props](https://styled-system.com/api#color) from
[Styled System](https://styled-system.com/).

## Component props

| Name   | Type                       | Default  | Required | Description                        |
| :----- | :------------------------- | :------- | :------- | :--------------------------------- |
| values | Array of Avatar            |          | ✓        | See below.                         |
| align  | `start`, `center` or `end` | `center` |          | Alignment of profile (responsive). |

### Avatar

This is a subset of
[Profile from `ProfilePreview`](/components/profile-preview#profile) plus `id`
and `href`:

| Name   | Type   | Default | Required | Description                                                                                                |
| :----- | :----- | :------ | :------- | ---------------------------------------------------------------------------------------------------------- |
| id     | String |         | ✓        | Identifier, e.g., the node ID generated by [Gatsby](https://www.gatsbyjs.org/).                            |
| avatar | Node   |         | ✓        | Avatar image (composition pattern).                                                                        |
| name   | Node   |         |          | Full name (composition pattern). You need to make sure that this component aligns with above `align` prop. |
| href   | String |         | ✓        | Path to which the avatar should link.                                                                      |

## Variants

`Avatars` does not support variants. If you customize
[Theme UI `Link`](https://theme-ui.com/components/link) via `styles.a`, this
will also affect all links in `Avatars`.
