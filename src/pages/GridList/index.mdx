# GridList

A list based on [CSS Grid](https://developer.mozilla.org/en-US/docs/Web/CSS/grid) to render components in a grid.

## Props

| Prop     | Description                             | Type                                                            | Required | Default  |
| :------- | :-------------------------------------- | :-------------------------------------------------------------- | :------- | :------- |
| `align`  | Alignment of entire grid.               | `oneOf(["left", "center", "right"])`                            | `false`  | `center` |
| `gap`    | Grid gap (horizontal and vertical)      | `oneOfType([number, arrayOf(number )])`                         | `false`  | `3`      |
| `render` | Render prop to render `values`.         | `func`                                                          | `true`   |          |
| `values` | Values to be displayed with `GridList`. | `arrayOf(object)`                                               | `true`   |          |
| `width`  | Width of grid values.                   | `oneOfType([number, arrayOf(number), string, arrayOf(string)])` | `true`   |          |

In addition `GridList` accepts [margin and padding props](https://rebassjs.org/props/#margin-and-padding). Each value in `values` needs to include an ID that is assigned to the `key` prop of the component used with `render`.

## Examples

### Avatars example

`GridList` delegates rendering of `values` to its `render` prop. This provides full control over what `GridList` can render. Objects in `values` must match destructured values of the `render` prop function argument, in this example `avatar` and `href`. We provide a `profiles` array to `values` and transform it with [`.map`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/map) to the format required by the `render` prop.
